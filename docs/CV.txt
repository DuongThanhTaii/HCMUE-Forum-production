#### Project Description

UniHub is a comprehensive university community platform I designed and built to address the fragmented communication landscape at HCMUE. Students and lecturers previously relied on scattered Facebook groups, Zalo chats, and email chains for academic collaboration. UniHub consolidates five core domains — a Stack Overflow-style discussion forum, a course-scoped learning resource center, a career job board, a real-time chat system with voice calls, and an AI assistant — into a single, integrated platform.

The system supports four user roles (Student, Lecturer, Moderator, Admin/SuperAdmin) with a granular role-based access control system, dual authentication providers (Azure AD SSO and local JWT), and real-time communication via WebSockets with Redis pub/sub for horizontal scaling.

- **Backend Development (.NET)**: Built the primary .NET 10 backend with MediatR CQRS pipeline (5 behaviors), EF Core persistence with repository and specification patterns, Result-based error handling, domain event dispatch via interceptors, and SignalR real-time hubs
- **Backend Development (NestJS)**: Built a secondary NestJS 11 backend with Prisma ORM, @nestjs/cqrs, Socket.IO gateways with Redis adapter, JWT/Passport authentication, and global maintenance mode guard
- **Frontend Development**: Developed the React 19 SPA with feature-sliced architecture, RTK Query for API state management (19 cache tag types), React Context for WebSocket state (chat, notifications, calls), and full bilingual support (Vietnamese/English)
- **Authentication & Security**: Integrated Azure AD SSO (MSAL popup) with automatic user provisioning, local JWT auth with refresh token rotation, multi-tier rate limiting, three-layer input validation, and MongoDB-backed user action logging
- **Real-Time Systems**: Implemented dual real-time protocols (SignalR + Socket.IO) with Redis backplane, chat with reactions/receipts/typing/presence/threading, WebRTC signaling for voice calls, notification push to user-specific rooms, and offline message queue with IndexedDB
- **DevOps**: Configured Docker Compose for local development, deployment pipelines for Vercel/Railway/Render, and an 8-seeder database initialization system

#### Key Achievements

1. **Designed a production-grade DDD modular monolith** with 7 bounded contexts, each with its own database schema — achieving true bounded context isolation at both the code and data layers

2. **Built a five-stage MediatR CQRS pipeline** (Validation → Logging → Performance → Exception → UnitOfWork) that provides cross-cutting concerns transparently to all 200+ command/query handlers

3. **Engineered a real-time chat system** with Socket.IO and Redis pub/sub supporting typing indicators, emoji reactions, read receipts, message threading, file sharing, voice messages, and WebRTC voice call signaling — handling reconnection, offline message queuing, and concurrent user presence tracking

4. **Implemented an offline-first chat architecture** using IndexedDB message outbox with configurable retry policy, automatically draining on three triggers: WebSocket reconnect, browser focus, and network online events

5. **Integrated dual authentication providers** (Azure AD SSO via MSAL popup + local JWT with bcrypt) with automatic user provisioning middleware and a thread-safe token refresh mutex preventing concurrent refresh calls

6. **Designed a four-layer authorization system**: frontend route guards → backend JWT/role guards → permission attribute checks → database-level scope values — with per-user and per-group permission overrides

7. **Built a complete admin observability stack** including runtime endpoint toggles, maintenance mode with admin bypass, MongoDB-backed user action audit logging with query interface, and configurable rate limiting across 5 policies

8. **Developed bilingual support** (Vietnamese/English) across 15 i18n namespace files per language with an automated verification script ensuring translation key parity

#### Technical Highlights

- **Event Sourcing** in the Learning module for document approval workflows, providing full audit trail with temporal queries
- **Shared Kernel** with `AggregateRoot<TId>`, `StronglyTypedId`, `Result<T>`, `Specification<T>`, `IRepository<T>`, and `IUnitOfWork` abstractions used consistently across all modules
- **Domain Event Dispatch** via `DomainEventInterceptor` — aggregate root events are automatically collected and dispatched through MediatR after `SaveChangesAsync`
- **RTK Query Tag Invalidation** — 19 cache tag types enable selective cache invalidation from WebSocket events, maintaining data consistency without full refetches
- **Polymorphic Report System** — a single `reports` table handles reports for both posts and comments via `reported_item_id` + `reported_item_type` columns
- **Recruiter Permission System** — per-company granular permissions (manage postings, review applications, update status, invite recruiters) at the `recruiters` table level
